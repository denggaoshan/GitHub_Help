My repository is marked as the wrong language

  • mac
  • windows
  • linux
  • all

GitHub uses our open-source Linguist library to determine file languages for syntax highlighting and repository statistics. Some files are hard to identify, and sometimes projects contain more library and vendor files than their primary code.

If you're receiving erroneous results, create an Issue on the Linguist repository, or contact support or your GitHub Enterprise site administrator for more help.

Automatically excluded files

Linguist excludes certain file names and paths from statistics. For a list of exclusions, take a look at the vendor.yml file.

Pull requests with new exclusion patterns are always welcome.

Markup rendering

Markup languages are rendered to HTML and displayed inline using our open-source Markup library.

At this time, we are not accepting new markup languages to show within GitHub. We are, however, actively maintaining our current markup languages. If you see a problem, please create an issue.
