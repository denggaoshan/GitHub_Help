Tips for configuring an A record with your DNS provider

  • mac
  • windows
  • linux
  • all

To set up a custom apex domain, you must configure an ALIAS, ANAME, or A records with your DNS provider, which may or may not be the same as your web hosting provider.

Warning: Do not create a CNAME record with your DNS provider for your custom apex domain! Doing so may cause issues with other services, such as email, on that domain.

For more information on custom apex domains, see "About custom domains for GitHub Pages sites."

Tip: You can only configure either a custom subdomain or a custom apex domain for a GitHub Pages site, unless you use a www subdomain.

Configuring an A record with your DNS provider

With your DNS provider, create A records that resolve to the following IP addresses:

  • 192.30.252.153
  • 192.30.252.154

To confirm that your A records have been set correctly, use the dig command:

dig example.com +nostats +nocomments +nocmd
;example.com
example.com.   73  IN  A 192.30.252.153
example.com.   73  IN  A 192.30.252.154

Configuring an ALIAS or ANAME record with your DNS provider

If your DNS provider supports ALIAS records (such as with DNSimple) or ANAME records (such as with DNS Made Easy), you may choose to create an ALIAS or ANAME record to username.github.io, instead.

Note: Some DNS providers allow an apex domain configured with an ALIAS record to point to some other domain name.

To confirm that your ALIAS or ANAME record has been set correctly, use the dig command:

dig example.com +nostats +nocomments +nocmd
example.com.     3600    IN A     199.27.XX.XXX

The IP address shown here should match the final IP address shown by dig username.github.io.

Configuring a www subdomain

If you configure both an apex domain (e.g. example.com) and a matching www subdomain (e.g. www.example.com), GitHub's servers will automatically create redirects between the two.

For example:

  • If your CNAME file contains example.com, then www.example.com will redirect to example.com.
  • If your CNAME file contains www.example.com, then example.com will redirect to www.example.com.

You may be able to use a custom subdomain other than www together with a custom apex domain via domain redirection (sometimes called "domain forwarding"). Note, however, that this will only work for User and Organization Pagesâ€”not for Project Pages.
