About custom domains for GitHub Pages sites

  • mac
  • windows
  • linux
  • all

There are two kinds of custom domains that can be used to redirect GitHub Pages sites: subdomains and apex domains.

Subdomains

A subdomain is configured with a CNAME record through your DNS provider.

We strongly recommend that you use a custom subdomain for these reasons:

  • It gives your GitHub Pages site the benefit of our Content Delivery Network.
  • It will not be affected by changes in the underlying IP addresses of GitHub's servers.
  • Pages will load significantly faster because Denial of Service protection can be implemented more efficiently.

Apex domains

An apex domain is usually configured with an A, ALIAS, or ANAME record through your DNS provider and is often assigned to one or more IP addresses.

Note: Some DNS providers support configuring apex domains with an ALIAS or ANAME record, but there is no industry standard for these. Some DNS providers (like DNSimple) allow an apex domain configured with an ALIAS or ANAME record to point to some other domain name.

We recommend using a custom subdomain instead of an apex domain for your GitHub Pages site.

How GitHub Pages sites use custom domains

Type of GitHub Pages site Host location on GitHub How the page is redirected Example custom domain
User Pages site username.github.io Automatically redirected to the custom domain that has been set for it user.example.com
Organization Pages site orgname.github.io Automatically redirected to the custom domain that has been set for it org.example.com
Project Pages site owned by a user account username.github.io/projectname Automatically redirected to a subdirectory of a User Pages site custom domain (user.example.com/projectname) as well as any custom domain specified by the user project.example.com
Project Pages site owned by an organization orgname.github.io/projectname Automatically redirected to a subdirectory of a Project Pages site custom domain (org.example.com/projectname) as well as any custom domain specified by the organization project.example.com

To troubleshoot problems with your custom domain, see "My custom domain isn't working."
