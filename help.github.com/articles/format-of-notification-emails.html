Format of notification emails

  • mac
  • windows
  • linux
  • all

Email notifications are delivered with the same HTML characteristics so that you can automatically filter or forward them as you desire.

Markdown, @mentions, emojis, links, and other special formats are automatically sent out as HTML. Here's an example of what you might see in your email client:

You can also opt to receive text-only emails, but you'll need to configure this setting on your mail client.

Unified mail headers

Emails are always sent with the following information:

  • The From address in all notification emails is notifications@github.com.
  • The To field in the email connects directly to the thread. That means that if you reply to the email, you'll actually be adding a new comment to the conversation.
  • If you're mentioned in a thread, or have already commented, we'll Cc you on the email. In all other cases, we'll Bcc you.

GitHub Enterprise will not add new comments to a conversation if you reply to its email notifications.

Shared read state

If you read a notification email, it'll automatically be marked as read in the Notifications section. An invisible image is embedded in each mail message to enable this, which means that you must allow viewing images from notifications@github.com in order for this feature to work.
